0x01 NFA&DFA

Finite Automata are recongnizers; they simple say “yes” or “no” about each possible input string

​ For example:

​ How to recognize the language of regular expression (a|b)*abb[1]

​ Let’s see the transition graph for an NFA:

​ We can see that theNFA，the right input well make a state transfer. When we inputabb,aabb,babb and so on, we will reach the 3 state, which is a double circle, means this state is accepting. And we input something like acabb or aba we can not reach here.

​ But there is a problem in this method to achieve this regular expression. In state 0, when we input a, we can not make sure the right next stat is 0 or 1. So we called this is a nondeterministic finite automaton.

In order to solve this problem, we can use the deterministic finite automaton, the DFA:
