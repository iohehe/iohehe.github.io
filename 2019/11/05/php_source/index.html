PHP数据传入点分析


0x01介绍

要验证PHP中污点传播漏洞大概需要验证三点，第一个是用户传入点，此点用户可控。 第二个是作用点， 此点触发了危险函数。第三个是在数据传播的路径上有没有效的过滤。

归根结底还是要使得传入危险函数的参数有害。这里讨论一下参数的传入。

  • key

用户通过交互web前端向服务器提交的内容通常先写在全局变量中。 有时我们只注意到了GET/POST/COOKIE等提交的变量的value, 而忽视了变量本身的key。

0x02 变量覆盖

变量覆盖是在脚本运行时对已有变量的非预期赋值。

  • extract

此函数在指定参数为EXTR_OVERWRITE或者缺省时，可以导致变量覆盖。

函数原型：

  • \$$

双$符表示PHP中的可变变量，可以构成PHP中比较独特的变量覆盖方式。一般发生在遍历一个传入的全局变量数组时：

  • parse_str

    此函数解析字符串中的QUERY STRING 如果没有设置第二个变量result则可能导致变量覆盖。

    函数原型：

  • mb_parse_str

    与parse_str类似，解析 GET/POST/COOKIE 数据并设置全局变量

  • import_request_variables

​ php.net原文中说明，将 GET／POST／Cookie 变量导入到全局作用域中。如果你禁止了 register_globals，但又想用到一些全局变量，那么此函数就很有用。(register_globals是PHP的一个特性。)

但是这个特性已经在php5.4.0移除。
