PHP 文件读写- 打开文件


PHP文件读取

PHP在处理文件时，通常包含三个步骤：打开文件->处理文件->关闭文件。

0x01 打开文件

打开一个文件，需要使用fopen函数:

  • 第一个参数：
  1. 绝对路径从根目录开始(linux->/,win通常在’c:\’)，相对路径描述一个相对于文档根目录的文件系统位置

  2. 为了路径表示简短，开发时通常会使用__DIR__,$_SERVER['DOCUMENT_ROOT']等进行路径拼接

  3. UNIX环境下，路径使用正斜线/，而在Windows下可以使用反斜线\,在fopen中，使用反斜线必须进行转义，即使用\\。

  4. 如果传入的文件路径是一个URL，如http://,ftp://等，PHP会根据URL打开一个protocol handler，这会带来一些问题。

  • 第二个参数：

    1. 文件模式：只读(r),读写(r+),只写(w),读写(w+),谨慎写(x),谨慎读写(x+),追加写(a),追加读写(a+),二进制(b),追加二进制(b+),文本(t)。

    2. r+和w+的区别在于，r+没有文件内容的清空权利和文件的创建权。而w+可以在文件不存在时创建这个文件，文件有内容则清空之(非追加模式)。

0x02 安全问题

打开文件时文件操作的第一步，如readfile或者file_get_contents这种直接传路径，而不是文件指针的函数也包含了打开文件这一步。此时需要的是一个资源路径，如果这个路径可控，常常会引发安全问题。

  • 任意文件读取

    如果打开文件功能设置了不正确的访问权限，可能造成任意文件读取漏洞。

  • 封装协议

    PHP带有很多内置URL风格的封装协议，可用于类似fopen()，copy(),file_exist()和filesize()的文件系统函数(参考)。可能造成SSRF,文件读取,Phar反序列化等问题。

  • 其他

    后续总结其他一些问题，如readfile函数直接将内容输出到显示器，可能造成一些xss等问题。: )
