0x01 认识编码

计算机是用二进制来认识世界的， 它只认识0和1两个数字。 符号是信息的载体， 不同的信息域使用着不同的符号体系。 这些符号在计算机中，对应着0和1。计算机需要处理这些符号，就要有一个映射体制，也就是编码。 在计算机世界中，需要一套规范，来制定各自的字符集，使得计算机使用者去遵守。字符集(Character Set)指定了字符和二进制的对应关系， 而编码就是将字符集转化为二进制的过程。码就是字，有了字就可写词，句子，文章。

0x02 编码种类

binary-to-text

ASCII， GB2312, GBK, Unicode, utf-8
ASCII: 为了解决英语世界到计算机世界的信息转化问题。用8位二进制也就是一个字节来表示字符，共128个。
后来使用各个字符集就开始了指定他们自己的编码规范。通常都是以人为规范的字节(8bit)为单位扩展的。
如中文这个字符集: GB2312 -> GBK -> GB18030
这时候，出现了很多编码规范， 需要一个统一的规范来使得计算机世界具有通用型， 于是需要制定一个统一的规范。就像每种字符集都由使用者制定了一个通往计算机世界的接口。 接口很多，每一种字符集需要从指定的接口进入， 字符集之间还需要转化。所以人们想制定一个统一接口。 在这种呼吁下，诞生了Unicode，也称统一码，万国码。
