Static Detection of Security Vulnerabilities in Scripting Languages

本文为Stanford大学2005年发表的文章追本溯源，这个Yichen Xie和Alex Aiken团队似乎是PHP安全静态分析比较早的，那个时候，台大的Huang等的WebSSARI应该出来了； Livshits等的Pixy也在构建中。 这一篇理论性的东西比较多，都是在构建数学模型，有借鉴意义。 再拿来总结一下。

本文提出的是一个PHP安全漏洞的静态分析算法， 他们提出了一种三层架构从块内，过程内， 过程间三个级别进行静态分析。 尝试解决脚本语言独有的一些的动态特性，如动态类型， 代码包含。

AST

AST: 将PHP解析成抽象语法树(用的PHP5.0.5)， 处理无封装语句作为main function, main function作为程序入口。略。

CFG

CFG: 单entry, 单exit, 基本的CFG生成算法。略。

Simulate

在语言建模的基础上，完成转化，生成每个块的summary（BlockSummary）

这个总的算法，一个块，以return和exit为出口， 对每一个基本块做一个simulate然后生成这个块的summary。

首先看对语言的建模:

  1. Type: 有字符，布尔，数组，静态无法确认的数据(如形参)。 这里没有对array等复合结构作处理。
  2. Const: 有字符，数字，true, false, null

  3. L-val: 左值，这里有variable x, 实参，array等复合结构

  4. Expr：表达式包含前两种， 另外还有表达式的一元操作二元操作。最后一个是type cast类型转换。

  5. Stmt: statement的话就是单一表达式给左值， 过程给左值， return, exit跟include。

    其中提到了，include这种statement是scripting language所特有的(which allows prorammers to dynamically insert code into the program)。

状态转化就是从L(第一行)到表示的值(最后一行)的映射过程。

来看值表示，在每一个function中从入口出找初始值Lo(范围是Loc)， 然后这些值可以表示为三类分别是(String, Boolean和Integer)， 其中，String是最基本的类型，也是我们关注的类型。 他由一些列的segment为元素组成，segment代表的可以是string, 也可以是a set of memory location（contains(o)）。

每个Lo就是一个Loc->Value的过程。有string, boolean, integer, Init-Values, unknow五种结果。(这个模型看着还是有点乱啊）。
