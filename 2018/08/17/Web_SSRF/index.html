#1. 概念
SSRF(Server-side request forgery)即服务器端请求伪造。
这张网图中，可以看到防火墙只允许外部用户访问系统中的web服务，而系统中右边那一堆其他服务显然是外部用户无法触及到的。此时的小黑人想要攻击这些从外网无法访问的内部系统。

攻击方式: 攻击者通过修改请求发起攻击，使服务器向第三方发起请求并返回。

Em….好像是这样

具体看个例子:

正常的服务流程是：提供一个正常的图片网址，服务器抓到然后返回给你。
注意此时的图片是服务器去请求的，也就是说在防火墙内部发起的。

那就可以绕过防火墙从内网发起请求访问内网资源。

问题发生在Url的解析上，要想办法访问localhost。如构造：
GET /?url=http://localhost/server-status HTTP/1.1 Host: example.com
这样子就可以访问web服务器可以接触到而外部无法接触的资源了。
那么利用的关键就是Url一方面能够绕过验证检查，一方面又能够解析成本地访问(想对内网攻击)。。。
